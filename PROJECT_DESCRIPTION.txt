WearAware — Sustainable Fashion Scanner
========================================

Author: Caitriona McCann
Started: November 2025
Last Updated: March 2026


OVERVIEW
--------

WearAware is a mobile application that helps users understand and reduce
the environmental impact of their clothing. Users scan clothing care labels
with their phone camera, and the app extracts fibre composition data,
calculates sustainability metrics (water usage, CO2 emissions, microplastic
shedding, chemical intensity), assigns a letter grade (A-F), and generates
an AI-powered sustainability summary. After scanning, users can browse
sustainable product alternatives and second-hand options. The app also
includes social community features, gamification, wardrobe management,
a community clothing marketplace, a trade and giveaway system with charity
shop dropboxes, and direct messaging between users.


ARCHITECTURE
------------

The system is a three-tier architecture deployed on a Proxmox LXC container:

  Tier         Technology                   Port     Role
  ------------ ---------------------------- -------- --------------------------
  Frontend     React Native / Expo (SDK 54) Expo Go  Mobile UI, camera, OCR
  Backend      Node.js 18 / Express         3000     REST API, business logic
  ML Service   Python Flask                 5000     CLIP model classification
  Database     PostgreSQL 14+               5432     Persistent storage
  Cache        Redis                        6379     In-memory caching
  LLM          Ollama (Llama 3.2 1B)        11434    AI sustainability summaries

Proxmox LXC container spec: 4 CPU cores, 8GB RAM, 50GB disk, no GPU.


HOW IT WORKS — END-TO-END FLOW
-------------------------------

1. AUTHENTICATION (src/context/AuthContext.js)

   - Firebase Authentication with three modes: email/password registration,
     email/password login, and anonymous guest mode
   - Auth state persisted via AsyncStorage so users stay logged in between
     sessions
   - Registered users are synced to the backend PostgreSQL users table via
     POST /api/users/sync
   - Guest users can scan and view results but cannot save scan history,
     post to the community feed, or access gamification, wardrobe or
     marketplace features
   - Password reset via Firebase sendPasswordResetEmail


2. SCANNING A GARMENT (src/screens/main/CameraScreen.js)

   The scan is a two-step process:

   Step 1 — Pre-Scan Form:
   - User enters brand name, selects item type (from 16 predefined types:
     Shirt, T-Shirt, Jeans, Dress, etc.), and chooses gender (Men's/Women's)
   - User can choose Full Scan mode (fibre composition + garment photo) or
     Care Instructions mode (care symbols only)

   Step 2a — Label Scan (OCR):
   - User photographs the clothing care label using Expo Camera or picks
     from gallery
   - The image is converted to base64 and sent to Google Cloud Vision API
     via src/services/visionApi.js
   - Vision API performs TEXT_DETECTION and LABEL_DETECTION
   - Extracted text is parsed by parseCareLabelText() which uses regex to
     identify:
       * Fibre composition — e.g. "60% Cotton, 40% Polyester"
       * Country of origin — e.g. "Made in Bangladesh"
       * Care instructions — washing symbols, temperature limits, drying
   - Connectivity is checked first via src/services/hybridOcr.js — if
     offline, the user is prompted to use manual input instead

   Step 2b — Garment Photo:
   - After the label scan, the user photographs the actual garment
   - The image is resized client-side (max 1200px) then uploaded as
     multipart form data via src/services/imageUpload.js
   - The backend strips EXIF/GPS data for privacy, generates a UUID
     filename, stores the image in a user-scoped directory and creates
     a thumbnail
   - The garment image URL is used by the CLIP service to generate visual
     attributes for the alternatives search query

   Alternative: Manual Input (src/screens/main/ManualInputScreen.js)
   - Users can manually enter fibre percentages and select fibre types
     from a picker modal (supports 30+ fibre types)
   - Same gender selector and item type picker as the camera screen


3. IMPACT CALCULATION (src/utils/impactCalculator.js)

   Once fibres are identified, the app calculates environmental impact
   using a research-based dataset covering 30+ fibre types:

   - Natural fibres: Cotton, Organic Cotton, Linen, Hemp, Jute, Bamboo,
     Sisal, Nettle, Pineapple Leaf, Banana Leaf, etc.
   - Animal fibres: Wool, Silk
   - Synthetic fibres: Polyester, Nylon, Acrylic, Spandex, Elastane
   - Regenerated fibres: Rayon, Viscose, Modal, Lyocell, Tencel

   Each fibre has data for:
   - Water usage (litres per kg) — ranges from 45L (Polyester) to
     125,000L (Wool)
   - CO2 emissions (kg per kg) — ranges from 0.03 (Modal) to 10.4 (Wool)
   - Biodegradability — boolean + decomposition time
   - Microplastic shedding (mg per kg per wash) — 0 for natural,
     1600-2200 for synthetics
   - Chemical intensity (0-100 scale)
   - Renewable energy potential (0-100 scale)
   - Water pollution (0-100 scale)

   The calculation:
   a) Multiplies each fibre's impact metrics by its percentage and the
      garment weight (looked up from the item_types table, default 300g)
   b) Computes a weighted sustainability score (0-100)
   c) Applies bonus/penalty modifiers based on microplastic, chemical,
      renewable energy and pollution thresholds
   d) Assigns a letter grade: A (80+), B (65-79), C (50-64), D (35-49),
      F (<35)

   This calculation exists in both the frontend (for immediate display)
   and backend (for authoritative storage).


4. SAVING THE SCAN (backend/routes/scans.js)

   - Scan data (fibres, brand, item type, weight, grade, score, metrics,
     raw OCR text, image URLs) is persisted to the PostgreSQL scans table
   - The backend independently recalculates environmental impact to ensure
     data integrity regardless of frontend values
   - User history cache is invalidated in Redis


5. AI SUMMARY (backend/services/aiService.js)

   - Ollama running Llama 3.2 1B locally on the server generates a 3-4
     sentence sustainability summary for each scan
   - The prompt includes grade, score, fibre composition, water usage and
     carbon footprint
   - Falls back to a template-based summary if Ollama is unavailable
   - Summaries cached in Redis for 24 hours (summary:{id})


6. RESULTS DISPLAY (src/screens/main/ScanResultScreen.js)

   The results screen has a tabbed interface:
   - Sustainability Tab: Hero grade indicator (A-F with colour), quick
     stats (water usage, CO2), fibre composition bars, AI-generated summary
   - Care Instructions Tab: Parsed care symbols with visual icons (wash,
     bleach, dry, iron, dry clean)

   Additional features:
   - Share to Feed — post the scan to the community with a caption
   - Edit Scan — modify brand, item type or fibre composition
   - Delete Scan — removed from database and history
   - Export — scan data exported as CSV, PDF or text report via
     src/services/exportService.js
   - Find Alternatives — navigates to AlternativesScreen


7. DETAILED BREAKDOWN (src/screens/main/BreakdownScreen.js)

   - Individual water footprint and carbon footprint per fibre
     (proportional to its percentage)
   - Biodegradability status and decomposition time
   - Irish context comparisons ("equivalent to X cups of tea",
     "equal to driving X km")


SUSTAINABLE ALTERNATIVES SYSTEM
---------------------------------

After scanning, users can browse alternatives on the Alternatives Screen
(src/screens/main/AlternativesScreen.js) with two tabs:

Tab 1: Web Results (Vertex AI Search)
   - Google Vertex AI Discovery Engine searches a curated data store of
     sustainable brand websites (Patagonia, Reformation, Pangaia, Tentree,
     People Tree, Everlane, Thought Clothing, etc.)
   - Authenticated via GCP service account (google-auth-library)
   - Search queries built from: gender + sustainable fibre alternative +
     CLIP visual attributes (colour, pattern, garment type) if a garment
     photo was taken
   - Product images extracted from Discovery Engine pagemap metadata
     (cse_image, og:image) — no additional HTTP requests needed
   - Fibre substitution logic: Polyester -> "organic cotton",
     Nylon -> "Tencel", Acrylic -> "merino wool", etc.
   - Results cached in Redis for 24 hours

Tab 2: Second Hand (eBay)
   - eBay Browse API searches for pre-owned clothing
   - OAuth2 client credentials flow with in-memory token caching
   - Filtered to clothing category, conditions: Used, Good, Very Good,
     Like New
   - Marketplace set to EBAY_IE for EUR pricing
   - Search queries also informed by CLIP visual attributes if available
   - Gender-aware queries (e.g. "womens navy blue t-shirt organic cotton")
   - Results cached in Redis for 24 hours


CLIP MACHINE LEARNING SERVICE (ml-service/)
---------------------------------------------

A Python Flask microservice running OpenAI's CLIP ViT-B/16 model from
HuggingFace Transformers:

Endpoints:
  /health                    GET   Health check, confirms model is loaded
  /extract-embedding         POST  Extract 512-dim normalised embedding
  /extract-embedding-batch   POST  Extract embeddings from multiple images
  /describe                  POST  Zero-shot garment classification

Image Classification (ml-service/clip_model.py):
The /describe endpoint uses prompt ensembling — classification
probabilities are averaged across 3 different text prompt templates per
label, providing 3-5% accuracy improvement over single-prompt
classification.

Categories:
  - Colours (34 labels): red, navy blue, sage green, off-white, burgundy,
    rust, denim blue, etc.
  - Patterns (19 labels): solid, striped, plaid, floral, leopard print,
    gingham, herringbone, etc.
  - Styles (13 labels): casual, formal, sporty, streetwear, bohemian,
    minimalist, etc.
  - Garment Types (35 labels): t-shirt, button-up shirt, puffer jacket,
    midi dress, dungarees, etc.

Image preprocessing: Images are centre-cropped to square before
classification to focus on the garment and reduce background noise.

Confidence thresholds: Attributes below 0.15 confidence are excluded
from search queries to avoid adding noise.


SOCIAL FEATURES
----------------

Community Feed (src/screens/main/FeedScreen.js)
  - Three feed modes: Following, Discover (all public scans), My Posts
  - Pull-to-refresh and infinite scroll pagination
  - User search to find and follow other users

Interactions (backend/routes/social.js)
  - Follow/Unfollow — directional relationships stored in follows table
  - Posts — share scans with caption and visibility (public/private)
  - Likes — toggle-based, unique per user per post
  - Comments — threaded (supports parent_id for replies)

Profile (src/screens/auth/ProfileScreen.js)
  - User stats: total scans, average grade, sustainability score
  - Privacy controls (public/followers/private)
  - Edit profile, view followers/following lists, profile picture


GAMIFICATION SYSTEM (backend/routes/gamification.js)
------------------------------------------------------

Achievements (15 predefined, categorised into scanning, social,
sustainability and streak):
  - Scanning: First Scan, Getting Started (5), Label Reader (25),
    Scanning Pro (100)
  - Sustainability: Green Choice (first A-grade), Eco Warrior (10
    A-grades), Water Saver (1000L saved), Carbon Cutter (10kg CO2 saved)
  - Social: Going Social (first share), Influencer (10 shares), Connected
    (first follow), Growing Community (10 followers)
  - Streak: Weekly Warrior (7-day scanning streak)
  - Wishlist: Conscious Shopper (5 wishlist adds)

Progress tracked incrementally. Achievements unlock automatically after
qualifying events and award points.

Challenges:
  - Weekly and monthly time-limited challenges
  - Users join challenges explicitly
  - Progress updates triggered by backend events

Leaderboard (src/screens/main/LeaderboardScreen.js):
  - Weekly, monthly and all-time rankings
  - Scored by sustainability score + achievement points
  - Cached in Redis (5-minute TTL)


WARDROBE AND OUTFIT PLANNING
------------------------------

Wardrobe (src/screens/main/WardrobeScreen.js)
  - Users import scanned clothing items into a personal wardrobe
  - Items organised by category: Tops, Bottoms, Dresses, Outerwear,
    Activewear, Accessories
  - Users can favourite items, log wears and list items on the marketplace
  - 2-column grid layout with sustainability grade badges

Outfits (src/screens/main/OutfitsScreen.js)
  - Users create named outfits from wardrobe items
  - Weekly planner assigns outfits to days of the week
  - Today's day is highlighted by default


COMMUNITY MARKETPLACE AND TRADE SYSTEM
----------------------------------------

Marketplace (src/screens/main/MarketplaceScreen.js)
  - Browsable feed of wardrobe items listed as free or for trade by
    other users
  - Filter by listing type: All, Free, Trade
  - Direct messaging to the item owner built in

Trade (src/screens/main/TradeScreen.js)
  - Users can offer wardrobe items as free giveaways or propose swaps
    with another user
  - Automatic charity shop dropbox allocation — the nearest charity shop
    between both users is selected
  - Each user receives a unique PIN and compartment number for the dropbox
  - Trade status: pending, accepted, completed, declined
  - Full trade lifecycle: create, respond, complete

Messaging (src/screens/main/MessagesScreen.js, ChatScreen.js)
  - Direct messaging between users accessible from profiles and the
    home screen
  - Conversation list with unread indicators and last message preview
  - Messages poll for new content every 5 seconds


CHARITY SHOPS MAP
------------------

  (src/screens/main/CharityShopsScreen.js)
  - Displays nearby charity and thrift shops on an interactive map
  - Uses expo-location for device position and react-native-maps for display
  - Distance filter: 2km, 5km, 10km, 25km

CACHING STRATEGY (backend/cache.js)
--------------------------------------

  Data                TTL         Key Pattern
  ------------------- ----------- ------------------------------------------
  Scan history        5 minutes   history:{uid}
  Item weights        1 hour      item_weight:{type}
  AI summaries        24 hours    summary:{id}
  Community feed      2 minutes   feed:{uid}
  Web search results  24 hours    websearch:{gender}:{type}:{fiber}:{hash}
  eBay results        24 hours    ebay:{gender}:{type}:{fiber}:{hash}
  Leaderboard         5 minutes   leaderboard:{period}

All cache operations fail silently (errors logged, never thrown) to
ensure the app degrades gracefully when Redis is unavailable.


DATABASE SCHEMA
----------------

Core tables (backend/database/schema.sql):
  - users — Firebase UID, email, timestamps
  - scans — brand, item type, fibres (JSONB), environmental score/grade,
    water usage, carbon footprint, care instructions, image URLs
  - item_types — 16 predefined garment types with estimated weights

Social tables (backend/database/social_schema.sql):
  - user_profiles — display name, bio, avatar, privacy, aggregate stats
  - follows — directional follower/following relationships
  - scan_posts — shareable scan entries with captions and visibility
  - likes, comments — engagement on posts (comments support threading)
  - achievements, user_achievements — definitions and per-user progress
  - challenges, user_challenges — time-limited challenges and participation
  - product_recommendations, wishlist — curated sustainable products
  - leaderboard — weekly/monthly/all-time ranking cache

DESIGN SYSTEM (src/theme/theme.js)
-------------------------------------

  - Light theme with soft off-white canvas (#F7F9F8) and deep forest
    green accents (#1A6B4A)
  - Dark mode supported via applyTheme('dark') with near-black canvas
    (#0E1412) and adjusted green (#49B386)
  - WCAG 2.1 AA compliant contrast ratios across all text levels
    (primary text 16.96:1, secondary 4.97:1, buttons 6.47:1)
  - Dynamic font scaling respecting system accessibility settings
    (capped at 1.5x via PixelRatio.getFontScale())
  - EU Accessibility Act compliant — 44dp minimum touch targets
    (WCAG 2.5.5), accessibilityRole and accessibilityLabel on all
    interactive elements across all screens
  - Grade colours: A (#00C96A green), B (#7ED957 lime), C (#FFD166 amber),
    D (#FF9F43 orange), F (#C0392B red)

DEPLOYMENT
-----------

All backend services run on a Proxmox LXC container:
  - 4 CPU cores, 8GB RAM, 50GB disk, no GPU
  - Node.js 18 backend on port 3000
  - Python Flask ML service on port 5000 (CLIP runs on CPU)
  - PostgreSQL and Redis on the same host
  - Ollama on port 11434

The frontend runs via Expo Go on the user's phone connected to the
same network as the server.
